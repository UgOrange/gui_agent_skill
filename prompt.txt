GUI Agent Skill 演示 Prompt 包

本文件提供可直接用于 Claude/Codex 的演示 prompts，目标是突出“可落地、低风险、可复现”的移动端自动化能力。

通用执行约束（建议所有演示都遵循）：
- 为了稳定复现，优先使用无状态：`execute --stateless`
- 每一步都设置超时：`--timeout-sec 45~90`
- 一次调用只做一个最小动作
- 仅基于最新 screenshot/caption 决策下一步
- 只读演示禁止点赞、评论、关注、发送消息、下单支付

Prompt 1：微信每日公众号趋势分析（只读）

你是 GUI 编排代理。请使用 GUI Agent Skill，以只读方式完成“每日微信公众号趋势扫描”。

目标：
- 采样文章并输出当日趋势摘要。

关键词：
- AI Agent
- 跨境电商
- 私域运营

执行流程：
1. 打开微信，进入搜索 / 公众号文章搜索入口。
2. 针对每个关键词，采样前 3-5 篇相关文章。skill 截图并返回
3. 你根据截图对每篇样本记录以下信息：
   - 公众号名称
   - 文章标题
   - 发布时间
   - 可见互动信号（阅读/点赞等，若可见）
4. 输出结构化总结：
   - 高频主题 Top 3
   - 标题常见写法
   - 后续可跟进的 3 个选题角度

约束：
- 全程只读，不进行点赞/评论/转发/关注。
- 每一步使用 stateless 模式。
- 每一步都带 timeout。
- 若界面有歧义，先停下并询问用户。

Prompt 2：小红书关键词内容调研（只读）

请使用 GUI Agent Skill 在小红书执行只读内容调研任务。

关键词：
- “通勤穿搭 春季”

目标：
- 采样 10 篇笔记并生成洞察总结。

执行流程：
1. 打开小红书并进入搜索。
2. 输入关键词并进入结果流。
3. 采样前 10 篇笔记，记录：
   - 标题
   - 作者
   - 可见点赞/收藏
   - 内容角度标签（通勤/学生/平价/极简等）
4. 输出：
   - 重复出现的主题
   - 内容形式分布
   - 3 个可复用选题

约束：
- 仅只读，不互动。
- 单步指令 + 截图校验。
- 优先使用 stateless。

Prompt 3：多平台商品自动比价（京东/淘宝/拼多多）

请使用 GUI Agent Skill 对同一商品进行跨平台比价。

商品：
- “iPhone 17 128G 国行 全新”

输出表格列：
- 平台
- 商品标题
- 到手价（如可见含券后）
- 店铺类型（自营/旗舰/个人）
- 预计时效
- 退换政策（如可见）
- 备注（规格不一致风险）

执行流程：
1. 依次打开京东、淘宝、拼多多。
2. 使用同一搜索词检索。
3. 进入可比商品详情并采集字段。
4. 排除不可比样本（激活机/翻新/非国行/规格不一致）。
5. 输出：
   - 最低价候选
   - 性价比最优候选

约束：
- 在下单/支付前停止。
- 若规格不明确，回到详情页二次确认。

Prompt 4：稳定直播演示脚本（3 步）

请使用 GUI Agent Skill 执行一个稳定的 3 步 live demo：
1. 先 `status`，确认设备在线和当前 app。
2. 再执行一次 `execute --stateless`，进入一个确定性页面。
3. 最后执行一次 `tap`（ratio 坐标）点击一个可预期按钮。

每一步都输出：
- 实际执行命令
- 核心返回字段：`success`、`session_mode`、`next_action`、`screenshot_path`
- 失败兜底动作（调整 timeout 后重试 1 次）

Prompt 5：微信竞品选题日监控（运营向）

请使用 GUI Agent Skill 模拟日常选题监控流程：
- 方向 A：AI 工具
- 方向 B：增长营销
- 方向 C：电商运营

任务：
1. 在微信文章搜索中分别检索三个方向。
2. 每个方向采样 5 篇。
3. 产出日报结构：
   - 方向热度排序
   - 新兴关键词
   - 建议发布时间窗口
   - 选题 shortlist

约束：
- 仅只读行为。
- 优先 stateless。
- 任何不可逆动作前先征求确认。

Prompt 6：坐标模式恢复演示（无模型规划）

当模型规划不稳定时，切换到直接坐标点击模式：
1. 用 `status` 确认当前上下文。
2. 用 `tap --coord-space ratio` 做确定性控制。
3. 每次只点击一步，逐步到达目标页面。

要求：
- 每一步说明坐标选择依据。
- 校验返回 `coordinate` 对象中的 `effective_coord_space` 和 `clamped`。
- 出现非预期页面时立即停止并询问用户。
